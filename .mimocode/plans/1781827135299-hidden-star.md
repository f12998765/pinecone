# 修复：切换数据源时图标闪烁

## 问题描述

从本地切换到 Linkding 时，会短暂显示本地图标，持续时间取决于网络状况。

## 根因分析

切换 `dataSource` 时的状态变化序列：

```
1. toggleDataSource() 设置 dataSource = 'linkding'     (同步)
2. $watch('dataSource') 触发 loadServices(true)         (同步)
3. loadServices() 更新 services                          (同步)
4. Alpine 批量刷新 DOM                                   (下一个微任务)
```

**关键发现**：步骤 1-3 都在同一同步执行中完成，Alpine 会批量处理 DOM 更新，所以不会有"旧 services + 新 dataSource"的可见帧。

**真正的闪烁来源**：
- `iconMap` 在 `init()` 时从 IndexedDB 加载并缓存了之前 linkding 会话的图标
- 切换到 linkding 后，对于 `iconMap` 中已有的域名，`getServiceIcon()` 立即返回缓存图标
- 对于 `iconMap` 中没有的域名，`getServiceIcon()` 返回 null → 回退到 `service.icon`（linkding 的 icon 为空）→ 回退到 `favicon.svg`
- 图标逐个通过 `IconFetcher.resolveDomains()` 异步解析，每个域名最多 4 秒超时
- 这导致图标"逐个弹出"的闪烁效果

**此外**：`toggleDataSource()` 没有在切换前清空 `services`，虽然 Alpine 批量更新理论上不会显示旧数据，但显式清空更安全。

## 修复方案

### 修改文件：`assets/js/app.js`

**改动 1：`toggleDataSource()` 切换前清空 services 和 iconMap**

```js
toggleDataSource() {
    this._resetRefreshState();
    this.services = [];          // ← 新增：立即清空，防止过渡期显示旧数据
    this.iconMap = {};           // ← 新增：清空图标缓存，防止显示旧图标
    this.dataSource = this.dataSource === 'local' ? 'linkding' : 'local';
    this.hideContextMenu();
},
```

**改动 2：`loadServices()` 中 linkding 分支，加载前先清空 services**

当前代码：
```js
if (this.dataSource === 'linkding') {
    if (this.linkdingData?.length > 0) {
        this.services = this._filterInvalid(this.linkdingData);
    } else {
        this.services = [];
    }
    return;
}
```

改为：先清空，再设置（确保过渡期无旧数据）：
```js
if (this.dataSource === 'linkding') {
    this.services = [];
    if (this.linkdingData?.length > 0) {
        this.services = this._filterInvalid(this.linkdingData);
    }
    return;
}
```

**改动 3：`syncLinkding()` 完成后，立即用已缓存的图标填充 iconMap**

当前代码（syncLinkding 末尾）：
```js
if (domains.size > 0) {
    IconFetcher.resolveDomains([...domains]);
}
```

改为：先从缓存加载已有图标，再异步解析剩余的：
```js
if (domains.size > 0) {
    const cached = IconFetcher.getAllCached();
    for (const [domain, url] of Object.entries(cached)) {
        if (domains.has(domain)) this.iconMap[domain] = url;
    }
    IconFetcher.resolveDomains([...domains]);
}
```

## 验证方式

1. 打开本地书签 → 切换到 Linkding → 确认无本地图标闪烁
2. 切换回本地 → 再切换到 Linkding → 确认图标平滑加载
3. 在 Linkding 模式下刷新图标 → 切换到本地 → 切换回 Linkding → 确认无旧图标
4. 移动端长按切换按钮测试
